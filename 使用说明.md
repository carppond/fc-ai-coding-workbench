# 拾光 AI 编程平台 — 开发与构建指南

## 环境要求

- Node.js >= 18
- Rust (通过 rustup 安装)
- Tauri CLI (`@tauri-apps/cli` 已包含在 devDependencies 中)

## 安装依赖

```bash
cd ai-coding-workbench
npm install
```

## 开发模式

启动开发服务器 + Tauri 窗口（热重载）：

```bash
npm run tauri dev
```

> 前端 Vite dev server 运行在 `http://localhost:1420`

单独运行前端（不启动 Tauri 窗口）：

```bash
npm run dev
```

## TypeScript 类型检查

```bash
npx tsc --noEmit
```

## Rust 后端编译检查

```bash
cd src-tauri
cargo check
```

## 生产构建

### 构建当前架构的 DMG

```bash
npm run tauri build
```

产物路径：`src-tauri/target/release/bundle/dmg/`

### 构建 Universal DMG（同时支持 M 芯片和 Intel）

1. 确保安装了双架构 Rust target：

```bash
rustup target add aarch64-apple-darwin
rustup target add x86_64-apple-darwin
```

2. 执行构建：

```bash
npm run tauri build -- --target universal-apple-darwin --bundles dmg
```

产物路径：`src-tauri/target/universal-apple-darwin/release/bundle/dmg/拾光 AI 编程平台_0.1.0_universal.dmg`

> 注意：`git2` 依赖需要 `vendored-openssl` feature 才能交叉编译 x86_64，已在 `Cargo.toml` 中配置。

### 仅构建 .app（不打包 DMG）

```bash
npm run tauri build -- --bundles app
```

## 其他说明

- 未配置 Apple Developer 证书时，DMG 未签名，其他 Mac 打开需要在「系统设置 → 隐私与安全性」中手动允许
- 前端源码在 `src/`，Rust 后端源码在 `src-tauri/src/`
- Tauri 配置文件：`src-tauri/tauri.conf.json`
